## 現在の姿勢

_get_reward の中だと、現在の姿勢は state_dict
  に入っていて、先頭でローカル変数に取り出し済みです（env.py:63, 65）。

  n_pendulum_rad, n_pendulum_vel = state_dict["n_pendulum_rad"], state_dict["n_pendulum_vel"]
  n_arm_rad,      n_arm_vel      = state_dict["n_arm_rad"],      state_dict["n_arm_vel"]

  この4つが「現在の姿勢」（離散化済み・ビン番号 0〜7）

  ┌────────────────┬─────────────────────────┬─────────────────────────────────┐
  │      変数      │          意味           │             理想値              │
  ├────────────────┼─────────────────────────┼─────────────────────────────────┤
  │ n_pendulum_rad │ 振子の角度 α のビン     │ n_best(=3.5) に近いほど直立     │
  ├────────────────┼─────────────────────────┼─────────────────────────────────┤
  │ n_pendulum_vel │ 振子の角速度 α̇ のビン   │ 中央(3.5)＝ほぼ静止             │
  ├────────────────┼─────────────────────────┼─────────────────────────────────┤
  │ n_arm_rad      │ アームの角度 θ のビン   │ n_arm_best(=3.5) に近いほど中央 │
  ├────────────────┼─────────────────────────┼─────────────────────────────────┤
  │ n_arm_vel      │ アームの角速度 θ̇ のビン │ 中央(3.5)＝ほぼ静止             │
  └────────────────┴─────────────────────────┴─────────────────────────────────┘

  → 報酬計算は、このビン番号と n_best の距離（例：abs(n_pendulum_rad -
  n_best)）を使って書くのが基本形です。

  ---
  連続値（生の角度・角速度）も使いたい場合

  離散化前の実数値も state_dict に入っています（env.py:89-92）。_get_reward
  の中では取り出していないので、必要なら state_dict["..."] で取れます。

  ┌────────────────────────────┬────────────────────────────────┬──────────┐
  │            キー            │              意味              │  理想値  │
  ├────────────────────────────┼────────────────────────────────┼──────────┤
  │ state_dict["pendulum_rad"] │ 振子角 α（ラジアン、連続値）   │ 0 が直立 │
  ├────────────────────────────┼────────────────────────────────┼──────────┤
  │ state_dict["arm_rad"]      │ アーム角 θ（ラジアン、連続値） │ 0 が中央 │
  ├────────────────────────────┼────────────────────────────────┼──────────┤
  │ state_dict["pendulum_vel"] │ 振子の角速度 α̇                 │ 0 が静止 │
  ├────────────────────────────┼────────────────────────────────┼──────────┤
  │ state_dict["arm_vel"]      │ アームの角速度 θ̇               │ 0 が静止 │
  └────────────────────────────┴────────────────────────────────┴──────────┘

  連続値を使うと「中央＝0」で距離が測りやすい（例：abs(state_dict["pendulum_rad"])
  が小さいほど直立）。ビンより滑らかな報酬を設計しやすいです。

  ーブルの行インデックス用なので、報酬計算には使いません。
